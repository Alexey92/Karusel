🎰 KARUSEL — Инструкция по эксплуатации

Общая структура системы

Адрес (объект): игровые автоматы с ESP32 + локальный сервер (мини-ПК на Windows).

ESP32 отправляют события на локальный сервер. Сервер записывает их в локальную базу данных (server/karusel.db) и сразу пересылает на глобальный сервер. Информация на публичных экранах формируется из данных локальной базы.

Офис: глобальный (облачный) сервер (Mac). Принимает данные от локальных серверов и отдельных ESP32 (для адресов, где один автомат).

Порядок администрирования

На локальном сервере

  1. В админ-панели создайте адрес и игровые автоматы. ID автоматов проставляйте подряд, начиная с 1.
  2. ESP32 для каждого автомата прошейте, заменив в строке 22 прошивки MACHINE_ID на реальный номер автомата.
  3. Номера автоматов в админ-панели должны соответствовать реальным MACHINE_ID в ESP32.

На глобальном (облачном) сервере

  1. Создайте адрес, соответствующий реальному объекту. Названия могут не совпадать — адреса сопоставляются по ключу (API Key).
  2. Автоматы можно создавать и редактировать вручную, но по умолчанию они создаются автоматически при поступлении первого события от локального сервера.

Windows (локальный сервер)

1. Автозапуск при включении ПК

Ярлык start_server.bat помещён в папку автозагрузки (shell:startup). Сервер запускается автоматически при старте Windows.

2. Ручной запуск

Дважды кликните start_server.bat в папке C:\karusel\.

3. Настройка start_server.bat для конкретного адреса

Откройте файл в Блокноте и пропишите:

set KARUSEL_CLOUD_URL=http://194.186.104.79:5050/api/event
set KARUSEL_CLOUD_KEY=ваш_api_key
set KARUSEL_LOCATION_ID=id_адреса

API Key и ID смотрите в глобальной админ-панели во вкладке «Адреса».

4. Публичный экран

Откройте два окна браузера:

http://192.168.1.100:5050/screen?monitor=0   (левый экран)
http://192.168.1.100:5050/screen?monitor=1   (правый экран)

Разнесите окна на разные мониторы, включите полноэкранный режим (F11).

5. Админ-панель (локальная)

Откройте браузер: http://localhost:5050/admin. Логин/пароль по умолчанию: admin / admin123. Рекомендуется сразу сменить пароль (вкладка «Настройки»).

6. Очистка локальной базы данных

Остановите сервер. Удалите файл C:\karusel\server\karusel.db. Запустите сервер снова — база создастся пустой.

Все локальные данные будут потеряны. Глобальная база не затрагивается.

macOS (глобальный сервер)

1. Запуск сервера

Откройте Terminal. Запустите скрипт:

cd ~/Documents/karusel/cloud
./start_cloud.sh

Или двойным кликом в Finder.

2. Админ-панель (глобальная)

На Mac: http://localhost:5050/admin. Из интернета: http://194.186.104.79:5050/admin.

3. Очистка глобальной базы данных

Откройте Terminal и запустите скрипт:

cd ~/Documents/karusel/cloud
./reset_db.sh

Или двойным кликом в Finder.

Все данные всех адресов будут удалены. Отменить невозможно.

Минимальный набор действий для запуска нового мини-ПК

1. Установка Python

Скачайте Python 3.12 с python.org. При установке отметьте галочку «Add Python to PATH». Проверьте: python --version.

2. Копирование проекта

Скопируйте папку karusel на мини-ПК в C:\karusel (можно с флешки).

3. Установка зависимостей Python

cd C:\karusel\server
python -m pip install -r requirements.txt

4. Настройка роутера на объекте

Подключитесь к роутеру, зайдите в 192.168.1.1. Задайте SSID karusel-net, пароль karusel2026 (пароль жёстко зашит в коде esp32, строка 20: изменение пароля только с перепрошивкой esp32). Закрепите IP мини-ПК как 192.168.1.100 через DHCP-резервацию.

5. Настройка start_server.bat

Откройте C:\karusel\start_server.bat в Блокноте:

set KARUSEL_CLOUD_URL=http://194.186.104.79:5050/api/event
set KARUSEL_CLOUD_KEY=ваш_api_key
set KARUSEL_LOCATION_ID=id_адреса

6. Автозапуск

Создайте ярлык для start_server.bat. Поместите его в shell:startup.

7. Первый запуск

Запустите start_server.bat. Откройте http://localhost:5050/admin. Войдите (admin/admin123). Создайте адрес и автоматы.

Настройка роутера в офисе (TP-Link TL-MR6400)

Роутер имеет статический IP 194.186.104.79.

  1. Подключите Mac к роутеру, узнайте его локальный IP: ipconfig getifaddr en0.
  2. Зайдите в роутер: 192.168.0.1, логин/пароль admin/admin.
  3. Перейдите в Forwarding → Virtual Servers → Add New.
  4. Заполните: Service Port 5050, Internal Port 5050, IP Address — IP Mac, Protocol ALL, Status Enabled.
  5. Нажмите Save.

Установка Python 3.12 и PostgreSQL на Mac

Python 3.12

  1. Скачайте установщик с python.org (macOS 64-bit installer).
  2. Запустите .pkg, следуйте инструкциям.
  3. Проверьте: python3.12 --version.

PostgreSQL

  1. Скачайте Postgres.app, откройте .dmg, перетащите в Программы.
  2. Запустите Postgres.app, нажмите Initialize.
  3. Откройте Terminal и выполните: 
    echo 'export PATH="/Applications/Postgres.app/Contents/Versions/latest/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
createdb karusel
psql -d karusel -c "CREATE USER karusel WITH PASSWORD 'karusel'; GRANT ALL PRIVILEGES ON DATABASE karusel TO karusel;"
psql -d karusel -c "GRANT ALL ON SCHEMA public TO karusel;"
  4. Установите зависимости Python: 
    cd ~/Documents/karusel/cloud
python3.12 -m pip install -r requirements.txt

Работа с PostgreSQL (командная строка)

Подключение

psql -d karusel

Просмотр таблиц

\dt

Адреса и ключи

SELECT id, name, api_key FROM locations;

Последние 20 событий

SELECT e.id, l.name AS адрес, m.name AS автомат, e.event_type, e.timestamp
FROM events e
JOIN locations l ON e.location_id = l.id
JOIN machines m ON e.machine_id = m.id
ORDER BY e.timestamp DESC LIMIT 20;

Очистка всех данных

DELETE FROM events;
DELETE FROM machines;
DELETE FROM jackpot_config;
DELETE FROM locations;

Сброс автоинкрементных счётчиков

ALTER SEQUENCE locations_id_seq RESTART WITH 1;
ALTER SEQUENCE machines_id_seq RESTART WITH 1;
ALTER SEQUENCE events_id_seq RESTART WITH 1;
ALTER SEQUENCE jackpot_config_id_seq RESTART WITH 1;

\q
