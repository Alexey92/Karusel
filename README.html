Часть 1. Как работает система

Из чего состоит система

КомпонентГде находитсяЗачем
ESP32Внутри каждого игрового автоматаЛовит сигнал выигрыша и отправляет на сервер
Мини-ПК (Windows)На объекте (в магазине)Принимает сигналы от ESP32, показывает публичный экран, хранит локальную статистику
Центральный сервер (Mac)В офисеСобирает данные со всех объектов, показывает общую статистику
РоутерНа каждом объекте и в офисеСвязывает ESP32 с мини-ПК, обеспечивает доступ в интернет

Как данные попадают в систему

  1. Игрок выигрывает приз → автомат выдаёт электрический сигнал.
  2. ESP32 внутри автомата ловит сигнал и отправляет его по Wi-Fi на мини-ПК.
  3. Мини-ПК записывает событие в локальную базу данных.
  4. Публичный экран (телевизор) обновляется — показывает свежую статистику.
  5. Раз в несколько секунд мини-ПК отправляет копию события на центральный сервер в офисе.
  6. В офисе можно открыть админ-панель и увидеть статистику по всем объектам.

Как связаны номера автоматов (ID)

В системе используются три уровня идентификаторов. Важно понимать их связь.

УровеньГде задаётсяПримерПояснение
MACHINE_ID Прошивка ESP32 1 Номер, зашитый в плате. Уникален в пределах одного объекта. Может повторяться на разных объектах.
local_id Таблица автоматов в облаке 1 Должен совпадать с MACHINE_ID на ESP32. Уникален в пределах одного адреса.
id (глобальный) Назначается автоматически 15 Сквозной уникальный номер во всей облачной базе. Пользователь его не задаёт и не видит в обычной работе.

Пример:

  • Объект «Колонтай»: ESP32 прошита с MACHINE_ID=1. В облаке для адреса «Колонтай» создан автомат с local_id=1.
  • Объект «Невский»: ESP32 тоже прошита с MACHINE_ID=1. В облаке для адреса «Невский» создан автомат с local_id=1.
  • Это два разных автомата, они не путаются, потому что привязаны к разным адресам.

2.1. Установка Arduino IDE (для прошивки ESP32)

  1. Скачайте Arduino IDE с arduino.cc.
  2. Установите как обычную программу.
  3. Откройте Arduino IDE.
  4. Файл → Настройки → в поле «Дополнительные ссылки для менеджера плат» вставьте: 
    https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json
  5. Инструменты → Плата → Менеджер плат. Найдите esp32 (Espressif Systems) → Установить.
  6. Инструменты → Управлять библиотеками. Найдите ArduinoJson (Benoit Blanchon) → Установить.

2.2. Установка Python на мини-ПК (Windows)

  1. Скачайте Python 3.12 с python.org.
  2. Запустите установщик.
  3. Обязательно поставьте галочку «Add Python to PATH».
  4. Нажмите Install Now и дождитесь завершения.

2.3. Установка зависимостей Python на мини-ПК (Windows)

  1. Откройте командную строку: Win+R → cmd → Enter.
  2. Выполните: 
    cd C:\karusel\server
python -m pip install -r requirements.txt
  3. Дождитесь надписи «Successfully installed».

2.4. Установка Python на центральный сервер (Mac)

  1. Скачайте Python 3.12 с python.org (выберите macOS 64-bit installer).
  2. Запустите установщик .pkg, следуйте инструкциям.
  3. Откройте Terminal (Finder → Программы → Утилиты → Terminal).
  4. Проверьте: python3.12 --version

2.5. Установка PostgreSQL на центральный сервер (Mac)

  1. Скачайте Postgres.app с postgresapp.com.
  2. Откройте .dmg, перетащите в Программы.
  3. Запустите Postgres.app, нажмите Initialize.
  4. В строке меню появится иконка слона — сервер запущен.
  5. Откройте Terminal и выполните: 
    echo 'export PATH="/Applications/Postgres.app/Contents/Versions/latest/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
createdb karusel
psql -d karusel -c "CREATE USER karusel WITH PASSWORD 'karusel'; GRANT ALL PRIVILEGES ON DATABASE karusel TO karusel;"
psql -d karusel -c "GRANT ALL ON SCHEMA public TO karusel;"

2.6. Установка зависимостей Python на Mac

  1. Откройте Terminal.
  2. Выполните: 
    cd ~/Documents/karusel/cloud
python3.12 -m pip install -r requirements.txt
  3. Дождитесь надписи «Successfully installed».

Часть 3. Первичная настройка

3.1. Запуск облачного сервера (Mac, офис)

  1. Убедитесь, что PostgreSQL запущен (иконка слона в строке меню).
  2. Откройте Terminal.
  3. Выполните: 
    cd ~/Documents/karusel/cloud
python3.12 main.py
  4. Должна появиться надпись [CLOUD] Сервер готов.
  5. Не закрывайте это окно.

3.2. Создание адреса в облачной админке

  1. На Mac откройте браузер: http://localhost:5050/admin
  2. Войдите: admin / admin123.
  3. Вкладка «Адреса» → введите название → «Добавить адрес».
  4. В таблице появится адрес и его API Key (длинная строка). Скопируйте ключ.

3.3. Настройка мини-ПК (Windows, объект)

  1. На мини-ПК откройте папку C:\karusel\.
  2. Найдите файл start_server.bat, правой кнопкой → «Изменить».
  3. Найдите и замените строки: 
    set KARUSEL_CLOUD_URL=http://194.186.104.79:5050/api/event
set KARUSEL_CLOUD_KEY=вставьте_сюда_скопированный_API_Key
set KARUSEL_LOCATION_ID=1
  4. Сохраните (Ctrl+S), закройте Блокнот.
  5. Запустите сервер: дважды кликните start_server.bat.
  6. Должна появиться надпись [SERVER] Сервер готов к работе.

3.4. Настройка роутера в офисе (TP-Link TL-MR6400)

Роутер уже подключён к интернету и имеет статический IP 194.186.104.79.

  1. Подключите Mac к роутеру (по Wi-Fi или кабелем).
  2. Узнайте IP Mac в локальной сети: 
    ipconfig getifaddr en0
    (если Wi-Fi) или en1 (если кабель). Запишите этот IP (например, 192.168.0.50).
  3. Откройте браузер, введите 192.168.0.1 (адрес роутера).
  4. Войдите: логин admin, пароль admin (по умолчанию; если меняли — ваш пароль).
  5. Перейдите в раздел Forwarding → Virtual Servers (или «Переадресация → Виртуальные серверы»).
  6. Нажмите Add New (Добавить).
  7. Заполните поля:
    • Service Port (Порт сервиса): 5050
    • Internal Port (Внутренний порт): 5050
    • IP Address: IP Mac из шага 2 (например, 192.168.0.50)
    • Protocol: ALL (или TCP)
    • Status: Enabled
  8. Нажмите Save.

Теперь облачный сервер доступен из интернета по адресу http://194.186.104.79:5050/admin.

3.5. Прошивка ESP32

  1. Откройте файл karusel_esp32.ino в Arduino IDE.
  2. В строках 31-35 измените настройки под ваш объект: 
    const char* WIFI_SSID = "karusel-net";
const char* WIFI_PASSWORD = "karusel2026";
const char* SERVER_URL = "http://192.168.1.100:5050/api/event";
const int MACHINE_ID = 1;  // Уникальный номер автомата (1, 2, 3...)
  3. Подключите ESP32 к ноутбуку USB-кабелем.
  4. В Arduino IDE: Инструменты → Плата → ESP32 Dev Module.
  5. Инструменты → Порт → выберите COM-порт.
  6. Нажмите кнопку «→» (Загрузить). Дождитесь надписи «Загрузка завершена».
Для каждого автомата меняйте MACHINE_ID (1, 2, 3...). На одном объекте не должно быть двух ESP32 с одинаковым номером.

Часть 4. Повседневные операции

4.1. Добавить новый адрес (объект) в облако

  1. Войдите в облачную админку: http://194.186.104.79:5050/admin.
  2. Вкладка «Адреса» → введите название → «Добавить адрес».
  3. Скопируйте появившийся API Key.
  4. На мини-ПК этого объекта отредактируйте start_server.bat (см. п. 3.3): вставьте ключ и укажите ID адреса.
  5. Перезапустите сервер на мини-ПК.

4.2. Добавить автомат в адрес

  1. В админке: вкладка «Адреса» → кнопка 🔧 у нужного адреса.
  2. Внизу заполните «Номер на объекте» (должен совпадать с MACHINE_ID в ESP32) и «Название автомата».
  3. Нажмите «Добавить».
Если автомат не добавлен вручную, он создастся автоматически при первом выигрыше с ESP32.

4.3. Изменить автомат (номер или название)

  1. Вкладка «Адреса» → кнопка 🔧 у нужного адреса.
  2. Измените номер или название прямо в полях ввода.
  3. Нажмите кнопку 💾 в той же строке.

4.4. Удалить автомат

  1. Вкладка «Адреса» → кнопка 🔧 у нужного адреса.
  2. Нажмите 🗑️ в строке автомата. Подтвердите.
Удаление автомата удаляет и все его события. Будьте осторожны.

4.5. Удалить адрес

  1. Вкладка «Адреса» → нажмите 🗑️ в строке адреса. Подтвердите.
Это удалит адрес, все его автоматы и все события. Отменить невозможно.

4.6. Перенести автомат на другой адрес

  1. Удалите автомат из старого адреса (п. 4.4).
  2. Добавьте автомат в новый адрес (п. 4.2) с тем же номером на объекте.

4.7. Настроить главный приз (джекпот)

  1. Вкладка «Адреса».
  2. В строке нужного адреса измените число в поле порога и нажмите ✓.
  3. Чтобы сбросить текущий счётчик — нажмите Сброс.

Джекпот срабатывает автоматически, когда счётчик достигает порога. Событие отмечается в истории как 🎰 ДЖЕКПОТ.

4.8. Сменить пароль администратора

  1. В админке: вкладка «Настройки».
  2. Введите старый пароль, новый пароль (дважды).
  3. Нажмите «Сохранить новый пароль».
Пароль должен быть не менее 4 символов. Не забудьте его!

4.9. Перезапустить локальный сервер (мини-ПК)

  1. Закройте чёрное окно с заголовком «KARUSEL Server».
  2. Дважды кликните ярлык «KARUSEL Server» на рабочем столе.
  3. Или просто перезагрузите мини-ПК — сервер запустится автоматически.

4.10. Очистить локальную базу данных (мини-ПК)

  1. Закройте сервер.
  2. Удалите файл C:\karusel\server\karusel.db.
  3. Запустите сервер снова — база создастся пустой.
Все локальные данные будут потеряны. Облачные данные не затрагиваются.

Часть 5. Командная строка на Mac (для системного администратора)

5.1. Полный сброс облачной базы данных

dropdb karusel
createdb karusel
psql -d karusel -c "GRANT ALL ON SCHEMA public TO karusel;"
# Перезапустите облачный сервер

5.2. Просмотр API-ключей всех адресов

psql -d karusel -c "SELECT id, name, api_key FROM locations;"

5.3. Сброс автоинкрементных счётчиков

psql -d karusel -c "ALTER SEQUENCE locations_id_seq RESTART WITH 1;"
psql -d karusel -c "ALTER SEQUENCE machines_id_seq RESTART WITH 1;"
psql -d karusel -c "ALTER SEQUENCE events_id_seq RESTART WITH 1;"

5.4. Смена пароля администратора

cd ~/Documents/karusel/cloud
python3.12 -c "
import sys; sys.path.insert(0, '../server')
from auth import hash_password
print(hash_password('новый_пароль'))
"
# Скопируйте вывод и выполните:
psql -d karusel -c "UPDATE users SET password_hash = 'скопированный_хеш' WHERE username = 'admin';"

5.5. Добавить адрес и автоматы напрямую в БД

# Генерация ключа
python3.12 -c "import secrets; print(secrets.token_urlsafe(32))"

# Добавление адреса
psql -d karusel -c "INSERT INTO locations (name, api_key) VALUES ('Новый адрес', 'сгенерированный_ключ');"
psql -d karusel -c "INSERT INTO jackpot_config (location_id) VALUES (ID_НОВОГО_АДРЕСА);"

# Добавление автоматов
psql -d karusel -c "INSERT INTO machines (local_id, name, location_id) VALUES (1, 'Автомат №1', ID_АДРЕСА);"
