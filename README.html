1. Краткое описание системы

Что это

Система фиксирует выигрыши с 10 игровых автоматов. Каждый автомат оснащён платой ESP32, которая при выигрыше отправляет HTTP-запрос на сервер. Сервер работает на мини-ПК внутри локальной сети магазина.

Компоненты

КомпонентОписаниеДоступ
Публичный экран Страница для телевизора. Показывает: выигрыши за 24 часа + последний приз. Открытый
Админ-панель Детальная статистика: по каждому аппарату, история, управление главным призом. По паролю
База данных SQLite (один файл karusel.db). Все события хранятся локально. Сервер
ESP32 × 10 Платы в автоматах. Ловят импульс, отправляют JSON на сервер. Wi-Fi

2. Перенос сервера на новый мини-ПК

2.1. Копирование файлов

Скопируйте папку karusel на новый ПК (например, в C:\karusel\).

Проверьте структуру:

C:\karusel\
├── schema.sql
├── README.html          ← этот файл
├── static\
│   ├── public.css
│   └── admin.css
├── templates\
│   ├── public.html
│   └── admin.html
├── server\
│   ├── main.py
│   ├── database.py
│   ├── models.py
│   ├── auth.py
│   └── requirements.txt
└── esp32\
    └── karusel_esp32.ino
Важно: Не копируйте файл karusel.db со старого ПК (если не нужно сохранять историю). База создастся автоматически при первом запуске.

2.2. Установка Python

Python нужен для запуска сервера. Устанавливается один раз.

  1. Скачайте Python с официального сайта: python.org/downloads
  2. Выберите версию 3.12.x (стабильная, проверенная).
  3. Запустите установщик.
  4. Обязательно поставьте галочку ✅ Add Python to PATH внизу первого окна!
  5. Нажмите Install Now и дождитесь завершения.
  6. Перезагрузите компьютер (рекомендуется).

Проверка установки

Откройте PowerShell и выполните:

python --version

Должно вывести: Python 3.12.x

Если команда не найдена — Python не добавлен в PATH. Переустановите с галочкой Add Python to PATH или пропишите путь вручную в переменных среды Windows.

2.3. Установка зависимостей

Откройте PowerShell в папке server:

cd C:\karusel\server
python -m pip install -r requirements.txt

Пойдёт установка пакетов: FastAPI, Uvicorn, aiosqlite, Jinja2, python-jose и других.

Дождитесь надписи Successfully installed ...

Если pip не найден — используйте: python -m pip install -r requirements.txt

2.4. Первый запуск

cd C:\karusel\server
python main.py

В консоли должно появиться:

==================================================
[SERVER] Запуск сервера KARUSEL...
[DB] Создаю новую базу данных...
[DB] База данных создана и заполнена тестовыми данными.
[AUTH] Установлен пароль администратора: admin123
[SERVER] Сервер готов к работе.
==================================================
INFO:     Uvicorn running on http://0.0.0.0:5050

Сервер работает, пока открыто это окно. Закроете — сервер остановится.

Откройте браузер и проверьте: http://localhost:5050/

2.5. Автозапуск при включении ПК

Чтобы сервер стартовал автоматически при включении мини-ПК, создайте .bat-файл и добавьте его в автозагрузку.

Шаг 1: Создайте файл

Создайте в папке C:\karusel\ файл start_server.bat с содержимым:

@echo off
cd /d C:\karusel\server
python main.py

Шаг 2: Добавьте в автозагрузку

  1. Нажмите Win + R
  2. Введите shell:startup и нажмите Enter
  3. Откроется папка автозагрузки
  4. Создайте ярлык для файла start_server.bat и поместите его в эту папку

После перезагрузки ПК сервер запустится автоматически. Окно PowerShell будет свёрнуто.

Важно: на мини-ПК должен быть настроен статический IP-адрес в локальной сети (через DHCP-резервацию в роутере). Именно этот IP прописывается в прошивках ESP32 в поле SERVER_URL.

4. Команды PowerShell (для администратора)

Все команды выполняются в отдельном окне PowerShell (сервер должен работать в другом окне).

4.1. Вход в систему (получение токена)

Токен действует 24 часа. Получите его один раз перед серией команд:

$r = Invoke-RestMethod -Uri http://localhost:5050/api/login -Method Post -ContentType "application/json" -Body '{"username":"admin","password":"admin123"}'
$token = $r.access_token

Теперь переменная $token содержит токен. Используйте её в следующих командах.

4.2. Эмуляция выигрыша (тестирование без ESP32)

Один выигрыш с автомата №1:

Invoke-RestMethod -Uri http://localhost:5050/api/event -Method Post -ContentType "application/json" -Body '{"machine_id":1}'

15 случайных выигрышей (для наполнения тестовыми данными):

1..15 | ForEach-Object {
    $m = Get-Random -Minimum 1 -Maximum 11
    Invoke-RestMethod -Uri http://localhost:5050/api/event -Method Post -ContentType "application/json" -Body "{`"machine_id`": $m}"
    Start-Sleep -Milliseconds 200
}
Write-Host "Готово!"

4.3. Просмотр статистики

Публичная статистика (без авторизации):

Invoke-RestMethod -Uri http://localhost:5050/api/public-stats

Статистика по одному аппарату (с авторизацией):

Invoke-RestMethod -Uri http://localhost:5050/api/admin/stats/1 -Headers @{Authorization="Bearer $token"}

Статистика по всем аппаратам:

Invoke-RestMethod -Uri http://localhost:5050/api/admin/stats -Headers @{Authorization="Bearer $token"}

4.4. Настройка главного приза

Посмотреть настройки джекпота для автомата №1:

Invoke-RestMethod -Uri http://localhost:5050/api/admin/jackpot/1 -Headers @{Authorization="Bearer $token"}

Установить порог джекпота (например, 50 выигрышей):

Invoke-RestMethod -Uri http://localhost:5050/api/admin/jackpot/1/threshold -Method Put -Headers @{Authorization="Bearer $token"} -ContentType "application/json" -Body '{"win_count":50}'

Сбросить счётчик в 0:

Invoke-RestMethod -Uri http://localhost:5050/api/admin/jackpot/1/reset -Method Post -Headers @{Authorization="Bearer $token"}

Установить счётчик вручную (например, 25):

Invoke-RestMethod -Uri http://localhost:5050/api/admin/jackpot/1/counter -Method Put -Headers @{Authorization="Bearer $token"} -ContentType "application/json" -Body '{"count":25}'

Установить всем автоматам одинаковый порог:

1..10 | ForEach-Object {
    $body = "{`"win_count`": 100}"
    Invoke-RestMethod -Uri "http://localhost:5050/api/admin/jackpot/$_/threshold" -Method Put -Headers @{Authorization="Bearer $token"} -ContentType "application/json" -Body $body
}
Write-Host "Порог 100 установлен для всех автоматов."

4.5. Просмотр истории

Последние 50 событий:

Invoke-RestMethod -Uri "http://localhost:5050/api/admin/events?limit=50&offset=0" -Headers @{Authorization="Bearer $token"}

Следующие 50 событий:

Invoke-RestMethod -Uri "http://localhost:5050/api/admin/events?limit=50&offset=50" -Headers @{Authorization="Bearer $token"}

4.6. Смена пароля администратора

Смена пароля пока доступна только через прямой доступ к БД. Для этого:

  1. Остановите сервер (Ctrl+C в окне сервера)
  2. Откройте файл C:\karusel\server\karusel.db программой DB Browser for SQLite
  3. Выполните в PowerShell генерацию хеша нового пароля:
python -c "
import hashlib, secrets
password = 'ваш_новый_пароль'
salt = secrets.token_hex(16)
salted = salt + password
hash_val = hashlib.sha256(salted.encode('utf-8')).hexdigest()
print(f'{salt}${hash_val}')
"

Скопируйте вывод и вставьте его в поле password_hash для пользователя admin в таблице users.

В будущих версиях добавим смену пароля прямо из админ-панели.

4.7. Полный скрипт «Вход + все проверки»

Скопируйте и выполните одним блоком — получите токен, проверите все автоматы и статистику:

# Вход
$r = Invoke-RestMethod -Uri http://localhost:5050/api/login -Method Post -ContentType "application/json" -Body '{"username":"admin","password":"admin123"}'
$token = $r.access_token
Write-Host "Токен получен.`n"

# Публичная статистика
Write-Host "=== ПУБЛИЧНАЯ СТАТИСТИКА ==="
Invoke-RestMethod -Uri http://localhost:5050/api/public-stats | Format-List

# Статистика по всем автоматам
Write-Host "`n=== СТАТИСТИКА ПО ВСЕМ АВТОМАТАМ ==="
$stats = Invoke-RestMethod -Uri http://localhost:5050/api/admin/stats -Headers @{Authorization="Bearer $token"}
$stats | ForEach-Object {
    Write-Host ("Автомат {0}: сегодня={1}, всего={2}, джекпот={3}/{4}" -f $_.machine_id, $_.wins_today, $_.wins_total, $_.jackpot_config.current_win_count, $_.jackpot_config.win_count_for_jackpot)
}

# Последние 5 событий
Write-Host "`n=== ПОСЛЕДНИЕ 5 СОБЫТИЙ ==="
$events = Invoke-RestMethod -Uri "http://localhost:5050/api/admin/events?limit=5&offset=0" -Headers @{Authorization="Bearer $token"}
$events.events | ForEach-Object {
    Write-Host ("[{0}] {1} — {2} — {3}" -f $_.id, $_.timestamp, $_.machine_name, $_.event_type)
}

Write-Host "`nГотово!"

7. Устранение неполадок

Сервер не запускается

  • Проверьте, что Python установлен: python --version
  • Проверьте, что зависимости установлены: python -m pip list
  • Проверьте, что файл karusel.db не повреждён. Если нужно — удалите его (создастся заново).
  • Убедитесь, что порт 5050 не занят другой программой.

Не открывается админка / публичный экран

  • Проверьте, что сервер запущен: откройте http://localhost:5050/
  • Нажмите Ctrl+Shift+R (жёсткая перезагрузка без кеша)
  • Проверьте консоль браузера (F12 → Console) на наличие ошибок

ESP32 не отправляет данные

  • Проверьте, что ESP32 подключён к WiFi (светодиод горит)
  • Проверьте правильность SERVER_URL в прошивке (IP мини-ПК + порт 5050)
  • Проверьте, что MACHINE_ID уникален для каждой платы (1-10)
  • Откройте Serial Monitor в Arduino IDE (115200 бод) и смотрите логи

Очистка базы данных (полный сброс статистики)

  1. Остановите сервер (Ctrl+C)
  2. Удалите файл: Remove-Item C:\karusel\server\karusel.db
  3. Запустите сервер снова — база создастся заново со стартовыми настройками
